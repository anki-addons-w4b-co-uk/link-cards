When you add cidd1234567890123 (card id) or nidd1234567890123 (note id) to a card or template this add-ons transforms the text to a clickable hyperlink. When you click it a preview of the card will open in an extra window or the note will be shown in an extra editor window. The latter option works for me so far but since it actually modifies a note it could in theory damage this note. You should be careful with very young add-ons without any ratings .... When you use such add-ons make sure to have backups and know how to restore them. This add-on also adds an entry to the browsre table context menu to copy the card id. For details about cids/nids see e.g. the end of this section from the manual. As always use it at your own risk. # mobile, ankimobile, ankidroid, ankiweb Add-ons only work on Windows, Mac, Linux. So on mobile you need a different solution. I use an Android phone and Ankidroid. I added some javascript to my back template that attaches all linked cards/notes at the bottom of my note on mobile, see here The review view (webview) can't access your database. It can only read media from your collection.media folder. This add-on adds an option to the main menu to export all linked cards/notes to the media folder so that my js function can read them from there. This solution for Ankidroid uses iframes and won't work on ankiweb and probably doesn't work on ankimobile. You would have to adjust the solution your back template for these platforms. # bugs etc. Please do not use reviews for bug reports or support requests.
